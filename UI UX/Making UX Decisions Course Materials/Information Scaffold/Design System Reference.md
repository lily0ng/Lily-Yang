I’ve included my go-to design systems. I’ve chosen these because they serve some of the most widely used digital products, are maintained by mature teams with serious budgets, and have great documentation.

| Name | Noteworthy |
| --- | --- |
| [Google’s Material Design](https://m3.material.io/) | See their rules on motion, dark theme and elevation |
| [Microsoft’s Fluent](https://www.microsoft.com/design/fluent/) | See their downloadable design toolkits |
| [Apple’s Human Interface Guidelines](https://developer.apple.com/design/human-interface-guidelines/guidelines/overview/) | See their rules for keyboard shortcuts |
| [IBM’s Carbon](https://carbondesignsystem.com/) | See their rules for colors in data visualization |
| [Shopify’s Polaris](https://polaris.shopify.com/) | See their rules on token naming and UX writing |
| [Gusto’s Workbench](https://workbench.gusto.com/) | See their robust component library |
| [Github’s Primer](https://primer.style/) | See their entire section on accessibility |
